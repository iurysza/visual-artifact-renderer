# Content types

High-level patterns for common artifact shapes. Each guide covers when to use it, which nodes fit, and how to organize information without drowning in style rules.

| Pattern | Use when | Primary nodes |
|---|---|---|
| [[architecture-diagrams\|Architecture diagrams]] | System topology, module relationships, runtime flows | `mermaid`, `grid`, `card` |
| [[dashboards\|Dashboards & metrics]] | KPIs, status boards, health overviews | `stat-card`, `status-grid`, `grid`, `table` |
| [[timelines\|Timelines & roadmaps]] | Sequences of events, phases, milestones | `timeline`, `section`, `card` |
| [[data-organization\|Data organization]] | Tables, comparisons, inventories, structured evidence | `table`, `data-table`, `comparison-table`, `status-grid` |
